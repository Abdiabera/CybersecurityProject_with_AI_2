By the end of this lesson you will be able to

  • Identify the difference between a single direct PC connection and a LAN connection through a router
  • Configure the Cloud-PT device in Packet Tracer to represent an ISP

Before you begin
Software needed
Cisco Packet Tracer (version 7.x or 8.x). Log in with your Cisco NetAcad account at netacad.com before starting.
Prior knowledge required
You must have completed the Peer-to-Peer lab. You should already know how to assign static IP addresses and run a ping test.
Important Packet Tracer note
The Cloud-PT device requires you to configure its DSL ports before connecting any cable to it. Skipping this step will cause a "cable cannot connect" error.

Network diagram
Your Packet Tracer topology
Part A — Single PC, direct path
PC3 connects directly to DSL Modem1, which connects to the ISP Cloud. There is no router in between. PC3 gets a public IP and is fully exposed to the internet.
Part B — Multiple PCs via router
PC4, PC5, and PC6 connect to Wireless Router0, which connects to DSL Modem0, then to the ISP Cloud. The router provides NAT, DHCP, and a firewall for the LAN devices.
Why two modems?
Each path to the ISP is independent. This is a classroom simulation showing two different types of home setups connecting to the same ISP — not one home with two modems.
Cloud0 = the ISP
The Cloud-PT device in Packet Tracer represents the ISP infrastructure — all the Tier 1/2/3 backbone you cannot see. Both modems connect into it using Phone cables on Modem4 and Modem5 ports.

Core vocabulary
Key concepts to know
DSL Modem
Modulator/Demodulator. Converts digital signals from your PC or router into analog signals the ISP telephone line can carry, and back again. It is the boundary between your home and the ISP.
Wireless Router (Home Router)
Connects multiple home devices to one internet connection. It provides NAT (hides private IPs), DHCP (assigns IPs automatically), and a basic firewall. It has a LAN side and a WAN side.
NAT — Network Address Translation
The router translates many private IPs (192.168.1.x) into one public IP before sending traffic to the internet. This hides your internal devices from outside attackers.
Public IP vs Private IP
Public IPs are routable on the internet (e.g. 203.0.113.1). Private IPs are only used inside your home network (e.g. 192.168.1.x). The router bridges these two worlds.
Phone cable in Packet Tracer
Used to connect DSL modems to the Cloud-PT device. It simulates the telephone line between your home and the ISP central office. Do not use a straight-through or crossover cable here.
Cloud-PT ports
Before connecting, you must add ports manually in Cloud0 → Config → DSL. Select Modem4 and click Add for the first modem. Select Modem5 and click Add for the second modem.
Default Gateway
The IP address of the device your PC sends traffic to when the destination is outside its own network. For LAN PCs, this is the router's LAN IP (192.168.1.1). For PC3 (direct), it is the modem-side IP.
Security posture
PC3 (Part A) has no router — it is directly exposed to the internet with a public IP. PC4–6 (Part B) sit behind a router. Same ISP, completely different attack surface. This is a key Security+ concept.
