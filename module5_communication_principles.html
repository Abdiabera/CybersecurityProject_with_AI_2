Module 5 at a glance
What we covered today
📡
Network Communication Models
The framework for data exchange
Models define how data is structured, transmitted, and received. They ensure devices from any manufacturer can communicate using the same rules.
🔗
Communication Protocols
Rules for every conversation
Protocols define method, language, timing, encoding, encapsulation, and confirmation — the six characteristics every protocol must address.
🧱
The OSI Model
7-layer reference framework
A conceptual model that breaks communication into seven precise layers. Used for understanding, designing, and troubleshooting networks worldwide.
🌐
The TCP/IP Model
4-layer practical model
The model that actually runs the internet. Four layers — Application, Transport, Internet, Network Access — each handling a specific function.
Key takeaways
1
OSI = theory. TCP/IP = practice. OSI gives you the language to troubleshoot. TCP/IP is what your data actually uses on the internet.
2
Protocols are strict rules — not suggestions. A device that does not follow the protocol does not get to communicate.
3
Every cyberattack targets a specific layer. Knowing the layers tells you exactly where the vulnerability is and how to defend it.
4
Standards make the internet universal. Without organizations like IETF and IEEE, every manufacturer would do things differently and devices could not communicate.
5
OSI mnemonic — All People Seem To Need Data Processing. Application · Presentation · Session · Transport · Network · Data Link · Physical.
Communication protocols
The rules that govern all network communication
📨
Message Format
How data is structured
Every message must follow a specific, predictable structure. Example: an email always has a subject, sender, recipient, and body. Wrong format = unreadable message.
📦
Message Size
Control data transmission
Protocols limit how large each data piece can be. Large data is broken into smaller packets — called segmentation. Only failed packets need resending — not everything.
⏱️
Timing
Speed and flow control
Controls when and how much data is sent. A packet arriving too late gets discarded. QoS prioritizes time-sensitive traffic like video calls over less urgent data.
📶
Encoding
Converting data for transmission
Data is converted into signals — electrical, light, or radio waves — for transmission. The receiver decodes the signals back into readable data. Wi-Fi uses radio waves.
✉️
Encapsulation
Adding addressing info
Every packet gets a header with source and destination addresses. Like a letter in an envelope — without the address, no one knows where to deliver it.
✅
Message Pattern & ACK
Confirmation requirements
TCP — sends and waits for an ACK before continuing. Reliable but slower. UDP — sends continuously without waiting. Fast but no delivery guarantee.
Why protocols matter for cybersecurity: Every attack exploits a protocol weakness — man-in-the-middle attacks exploit authentication protocols, DDoS attacks exploit connection protocols, packet injection attacks exploit data formatting. Understanding protocols means understanding where networks are vulnerable.

OSI vs TCP/IP — Layer Mapping
OSI Model — 7 Layers (Theory)
Layer 7 Application
Layer 6 Presentation
Layer 5 Session
Layer 4 Transport
Layer 3 Network
Layer 2 Data Link
Layer 1 Physical
TCP/IP Model — 4 Layers (Practice)
Application Combines OSI Layers 5, 6, 7
Transport Same as OSI Layer 4
Internet Same as OSI Layer 3
Network Access Combines OSI Layers 1 & 2
Key distinction: OSI = use it to think and troubleshoot (7 precise layers). TCP/IP = use it to work and configure (4 practical layers). Both live in your head simultaneously as a network professional.

RFCs — Request for Comments
📄
RFCs are the official technical documents of the internet — published and managed by the IETF
791
RFC 791 — defined IPv4 — the addressing system the internet ran on for decades
793
RFC 793 — defined TCP — written in 1981, still governs TCP today
DNS
RFC 1034 & 1035 — defined DNS — how domain names resolve to IP addresses
Security+ exam tip: Know which organization owns which domain. IETF = protocols. IEEE = hardware. ICANN = domain names. IANA = port numbers. ITU = global telecom. TIA = physical infrastructure.
